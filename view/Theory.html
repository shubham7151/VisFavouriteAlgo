1. Introduction

The Edmonds-Karp algorithm is an implementation of the Ford-Fulkerson method for computing the maximum flow in a flow network. It uses breadth-first search (BFS) to find augmenting paths. The algorithm runs in O(VE2) time, where V is the number of vertices and E is the number of edges in the graph.

2. Flow Network

A flow network is a directed graph where each edge has a capacity and each edge receives a flow. The amount of flow on an edge cannot exceed the capacity of the edge. The goal is to maximize the total flow from a source vertex (S) to a sink vertex (T).

3. Residual Graph

The residual graph is used to find augmenting paths in the flow network. For each edge in the original graph, the residual graph contains two edges: one in the same direction with a capacity equal to the remaining capacity of the original edge, and one in the opposite direction with a capacity equal to the flow on the original edge.

4. Edmonds-Karp Algorithm

The Edmonds-Karp algorithm repeatedly finds augmenting paths using BFS and augments flow along these paths until no more augmenting paths can be found. Each BFS finds the shortest augmenting path in terms of the number of edges.

5. Applications

The Edmonds-Karp algorithm is used in various applications, including:

  • Network flow optimization
  • Matching problems in bipartite graphs
  • Circulation problems
  • Connectivity in networks

6. Pseudo Code

    function BFS(source, sink, parent):
        Create a queue, Q
        Mark all nodes as not visited
        Enqueue source and mark it as visited
        while Q is not empty:
            u = Q.dequeue()
            for each edge (u, v) in residual graph:
                if v is not visited and capacity of edge (u, v) > 0:
                    Enqueue v and mark it as visited
                    parent[v] = u
                    if v == sink:
                        return true
        return false
    
    function edmondsKarp(source, sink):
        Initialize flow in all edges to 0
        while BFS(source, sink, parent) is true:
            path_flow = Infinity
            s = sink
            while s != source:
                path_flow = min(path_flow, capacity[parent[s]][s])
                s = parent[s]
            v = sink
            while v != source:
                u = parent[v]
                flow[u][v] += path_flow
                flow[v][u] -= path_flow
                v = parent[v]
            max_flow += path_flow
        return max_flow
